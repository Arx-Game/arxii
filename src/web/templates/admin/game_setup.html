Welcome to a new Arx-based instance

This is the starting point for configuring your game. The flow is:

  1. Seed defaults — Load sane defaults populates a baseline playable game (rules + character-creation content). Idempotent; never overwrites your edits.
  2. Author content — add or edit species, paths, magic, combat, traits, etc. via the World apps on the admin index.
  3. Build the world — World Builder digs rooms and areas directly on the shared map canvas (staff-only; area tree, drag-to-place, promote-to-authored).
  4. Load private content repo — {% if content_repo_configured %} Load content repo builds and upserts entries from your CONTENT_REPO_PATH checkout. Safe to re-run; updates existing rows by natural key.
    Export to content repo writes authored content models from the DB back to the CONTENT_REPO_PATH checkout's fixtures/ directory.
    Push to lore repo commits and pushes exported fixtures to the content repo's origin main. {% else %} set CONTENT_REPO_PATH in src/.env to enable this step. Until then, the Import Data upload above remains the path for loading ad-hoc fixture files. {% endif %}
  5. Tune mechanics — Game Tuning difficulty analytics + simulation.
  6. Monitor the live game — Game Ops progression, economy, story/GM, and reports-queue analytics.
  7. Audit vow suitability — Sphinx of Black Quartz shows which anchor-role vows are swearable today, per Tradition.
  8. Export your game — Export Data saves your configuration as a portable JSON fixture; Import Data loads one.
