# ADR-0101: House aspects are catalog-only

**Status:** Accepted (2026-07-07, ApostateCD ruling on #2079)

A house claim's regional aspects (patron deity, house virtue, vice, totem…) are
answered ONLY by picking from an authored `HouseAspectOption` catalog — there is
no free-text answer path, and none should be added. The normalized list is the
thematic guardrail: given free text, players author options that undermine
regional theme while nominally satisfying it (the "bloodthirsty white hats via a
self-authored 'Divine Vengeance' virtue" failure). The fence is the absence of
the off-theme option, not reviewer taste; staff review then only judges whether
the freeform prose (backstory, words, sigil) lives up to the picks. Freeform
stays confined to the stylings, where divergence is cosmetic. A normalized
option row is also what future mechanics key off (geas enforcement, ledger
surfaces) — prose never could be. **Rejected alternative:**
pick-or-author-with-staff-approval — it converts every review into a taste
argument and lets covers accumulate. Catalog gaps are fixed by authoring new
options, not by opening text input.
