# Clash Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the Clash mechanic — multi-round contested struggles in combat across five flavors (Clash, Suppress, Break Free, Ward, Break), with anima-Strain as the lever.

**Architecture:** A `Clash` model (discriminator over four meter shapes) lives in `world/combat`. Each round, PC contributions run through the existing `use_technique` cast pipeline in "clash-commit mode" (a custom `resolve_fn` that captures the `CheckOutcome` instead of applying damage), convert to progress deltas, aggregate, and move a meter; threshold crossings resolve through authored consequence pools. `resolve_round` gains a clash post-pass mirroring the existing `_resolve_declared_challenges` post-pass. The new Strain mechanism is built general but consumed only by Clash in this plan.

**Tech Stack:** Django / Evennia, PostgreSQL, `SharedMemoryModel`, FactoryBoy, DRF. Source spec: `docs/superpowers/specs/2026-05-22-clash-design.md`.

## Conventions for every task

- **TDD.** Every task: write the failing test → run it, confirm red → implement minimally → run it, confirm green → commit. Use the `superpowers:test-driven-development` discipline.
- **Tests:** `arx test` only. Targeted form: `arx test world.combat.tests.test_clash_models -k test_name`. Never `pytest`, never `python -m`.
- **Migrations:** `arx manage makemigrations combat` (and `magic` where noted) — uses the project's custom `makemigrations`. Apply with `arx manage migrate`. Consolidate: generate the combat migration **once** at the end of Phase 1, not per-model.
- **Models:** all concrete models subclass `SharedMemoryModel` (`from evennia.utils.idmapper.models import SharedMemoryModel`). No `JSONField`. No `Meta.ordering` unless sequential. Constants/TextChoices go in `src/world/combat/constants.py`.
- **Type annotations:** annotate all new functions (args + returns). Run `ruff check` on touched files; the pre-commit hook enforces ANN rules.
- **Combat is flat-file:** models → `src/world/combat/models.py`, services → `src/world/combat/services.py`, types/dataclasses → `src/world/combat/types.py`, constants → `src/world/combat/constants.py`, factories → `src/world/combat/factories.py`, serializers → `src/world/combat/serializers.py`, admin → `src/world/combat/admin.py`. Tests are flat files in `src/world/combat/tests/test_clash_*.py` — there is **no** `tests/integration/` subdir.
- **Commits:** one per task, present-tense conventional style (`feat(clash): ...`, `test(clash): ...`). Stage only the files the task touches — never `git add -A`.
- **Branch:** `clash-design` (already created; spec + positioning notes already committed there).
- **Pattern references** below point to exact `file:line` from the codebase map — open them and mirror the pattern rather than inventing a new one.
