## Spec â€” Origin-story templates: per-beginning structured background (#2478)

**Status:** spec-review Â· **Branch:** `feature-2478-origin-story-templates-per-beginning-str` Â· **Date:** 2026-07-18

### Goal

Replace the free-text CG Background textarea with a guided, template-driven origin
story. Each Beginning fixes a narrative **frame** (e.g. "every Khati story is an escape
from Salvation"); the player authors the **slots** inside it (who/why/how/who-helped/
what-was-left-behind). The result is assembled prose written to `Profile.background`
(preserving all existing consumers) **plus** queryable structured slot rows for future
GM/story tools. Deferral is first-class (nothing, slots-only, or complete), resumable
post-CG from the character sheet â€” mirroring #2427's Glimpse contract exactly. No LLM.
All authored text lives in arx2-lore via the content pipeline.

### User Stories

1. As a **player**, I want my origin story guided by my beginning's frame rather than a
   blank textbox, so that I write within a coherent narrative structure instead of
   staring at an empty field.
2. As a **player**, I want prompted slots (who helped, what was left behind) with
   examples, so that I cover the story beats the world expects without guessing.
3. As a **player**, I want to defer my origin story and finish it later from my
   character sheet, so that I'm not blocked at CG when I haven't decided all the
   details yet.
4. As a **player**, I want my assembled origin story visible as my background prose,
   so that it reads naturally wherever my background is displayed.
5. As a **GM/staff**, I want structured slot rows I can query, so that story tools can
   surface "who helped this character escape" or "what was left behind" without parsing
   free-text prose.
6. As a **staff author**, I want to author template frames and slot prompts in the lore
   repo, so that all narrative content is SSOT-governed and round-trips through the
   content pipeline.
7. As a **developer**, I want the pattern to mirror #2427's Glimpse flow exactly, so
   that there is one deferral/finish-later shape in the codebase, not two.

### Decisions (user-ratified)

6. **Deferral mirrors #2427 exactly.** State cached on a `origin_story_state` field,

### Design

#### Models (`src/world/character_creation/models/origin_story.py` â€” new file)

Three new models. Content models ride the lore-repo content pipeline (NaturalKeyMixin
+ CONTENT_MODELS); instance data does not.

**1. `OriginTemplate`** â€” authored catalog (content model).

- `beginning` â€” FKâ†’`Beginnings` (CASCADE). Multiple templates per beginning allowed;
  today one active template auto-assigns.
- `name` â€” CharField(100). Part of natural key.
- `frame_narrative` â€” TextField. The fixed frame prose every character with this
  beginning shares (e.g. "Your story begins with escape from Salvation...").
- `is_active` â€” BooleanField(default=True).
- `sort_order` â€” PositiveSmallIntegerField(default=0).
- `NaturalKeyConfig.fields = ["beginning", "name"]` â€” mirrors `Beginnings`'s
  `["starting_area", "name"]` shape. Dependencies: `["character_creation.Beginnings"]`.
- Registered in `CONTENT_MODELS` as `"character_creation.origintemplate"`.
- `Meta.unique_together = [["beginning", "name"]]`.

**2. `OriginTemplateSlot`** â€” authored slot prompts (content model).

- `template` â€” FKâ†’`OriginTemplate` (CASCADE).
- `name` â€” CharField(100). Part of natural key (e.g. "Who helped you escape?").
- `prompt` â€” TextField. The question shown to the player.
- `example` â€” TextField(blank=True). Illustrative answer.
- `sort_order` â€” PositiveSmallIntegerField(default=0).
- `is_required` â€” BooleanField(default=True).
- `NaturalKeyConfig.fields = ["template", "name"]` â€” mirrors `BeginningTradition`'s
  `["beginning", "tradition"]` shape. Dependencies:
  `["character_creation.OriginTemplate"]`.
- Registered in `CONTENT_MODELS` as `"character_creation.origintemplateslot"`.
- `Meta.unique_together = [["template", "name"]]`.

**3. `CharacterOriginSlot`** â€” instance data (NOT a content model, never exported).

- `sheet` â€” FKâ†’`character_sheets.CharacterSheet` (CASCADE).
- `slot` â€” FKâ†’`OriginTemplateSlot` (PROTECT).
- `value` â€” TextField. The player's authored answer.
- `Meta.unique_together = [["sheet", "slot"]]`.
- Mirrors `CharacterGlimpseTag` (`glimpse.py:65-88`).

**4. `CharacterSheet.origin_story_state`** â€” new field on the existing model.

- `origin_story_state` â€” CharField(max_length=20,
  choices=OriginStoryState.choices, default=OriginStoryState.NOT_STARTED).
- Cache of truth â€” maintained by the service layer, never written directly. Mirrors
  `CharacterAura.glimpse_state` (`aura.py:60-69`).
- `OriginStoryState` TextChoices in `character_creation/constants.py`:
  `NOT_STARTED` / `SLOTS_ONLY` / `COMPLETE`.

#### Service layer (`src/world/character_creation/services/origin_story.py` â€” new)

Mirrors `world/magic/services/glimpse.py`. Single write path so
`origin_story_state` never drifts from the slot rows.

- `refresh_origin_story_state(sheet) -> OriginStoryState` â€” recompute from
  `CharacterOriginSlot` rows + assembled-prose presence. SLOTS_ONLY if any slots
  filled but prose not assembled; COMPLETE if all required slots filled and prose
  written; NOT_STARTED otherwise. Mirrors `refresh_glimpse_state` (glimpse.py:28-39).
- `set_origin_slot(sheet, slot, value)` â€” upsert `CharacterOriginSlot`, recompute state.
  Mirrors `set_glimpse_tags`.
- `clear_origin_slot(sheet, slot)` â€” delete the row, recompute state.
- `assemble_origin_prose(sheet) -> str` â€” compose frame narrative + slot answers into
  prose. Pure template concatenation (no LLM). Called at finalize and on sheet-editor
  save.

#### Finalize wiring (`_apply_sheet_demographics` in `character_creation/services.py`)

Extends the existing demographics function (currently `services.py:418-419` writes
`draft_data["background"]` verbatim). New keys consumed from `draft_data`:

- `origin_template_id` â€” the auto-assigned template (resolved from
  `draft.selected_beginnings` if not explicitly set).
- `origin_slots` â€” dict mapping slot PK â†’ player's answer.

At finalize: resolve the template, upsert `CharacterOriginSlot` rows from
`origin_slots`, call `assemble_origin_prose` and write the result to
`Profile.background` (replacing the current verbatim write), then
`refresh_origin_story_state`. The old `draft_data["background"]` free-text key is
still read if present (backward compat for in-progress drafts created before this
feature ships).

#### CG API (`src/world/character_creation/views.py`)

- `CGOriginTemplateViewSet` (ReadOnlyModelViewSet) â€” `GET /api/character-creation/
  origin-templates/?beginning=<id>`. Returns active templates for a beginning, each
  with nested slots. `pagination_class = None` (ADR-0138 opt-out, same as every CG
  ViewSet). Mirrors `CGGlimpseTagViewSet` (`views.py:505-510`).
- Today one active template auto-selects; player-selection UI deferred (the endpoint
  returns a list, so no schema change when #2470 prologue variants land).

#### Post-CG API (`CharacterSheetViewSet` extension)

- `POST /api/character-sheets/{id}/set-origin-slot/` â€” body `{slot_id, value}`.
- `POST /api/character-sheets/{id}/clear-origin-slot/` â€” body `{slot_id}`.
- Dispatch the service-layer functions; gated by sheet ownership (same
  `X-Character-ID` / ownership contract as the Glimpse actions).

#### Frontend â€” CG (`frontend/src/character-creation/components/OriginStorySection.tsx`)

- Replaces the free-text `<Textarea>` at `IdentityStage.tsx:188-204`.
- Fetches template via `useOriginTemplates(draft.selected_beginnings?.id)`.
- Renders the `frame_narrative` as read-only prose, then each slot as a labeled
  `<Textarea>` with prompt + example.
- Slot answers write to `draft_data["origin_slots"]` immediately via `useUpdateDraft`
  (same immediate-write pattern as GlimpseSection tag picks).
- Graceful fallback: if no active template exists for the beginning, falls back to the
  old free-text textarea (same graceful-empty-catalog behavior #2427 established).
- Frame + slots always visible (no collapse-by-default).

#### Frontend â€” Character sheet (`frontend/src/character_sheets/components/OriginStoryEditorDialog.tsx`)

- Same guided flow, mounted on the sheet. Reads `CharacterOriginSlot` rows, writes
  through the sheet API actions, reassembles prose on save.
- Visible only when `can_finish_origin_story` is true in the sheet payload
  (privileged-only flag, same as `can_finish_glimpse`).
- Shares a presentational `<OriginSlotFields>` component with the CG mount (renders
  slots from a template + values); the two mounts differ only in persistence (draft
  PATCH vs. sheet API).

#### Stage validation (`character_creation/validators.py`)

The origin story is **not required** at CG submit â€” mirroring #2427 exactly, where
the Glimpse has no validation gate and flows into `finalize_magic_data` unconditionally
with empty defaults (`services.py:1153-1174`). The IDENTITY stage validator
(`get_identity_errors`) does not check origin slots. A player may leave all slots empty
and submit; the origin story is finished post-CG from the sheet editor. This means no
deferral flag or `defer_parents`-style boolean is needed â€” the absence of slot data IS
the deferred state (`origin_story_state=NOT_STARTED`).
